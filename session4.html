🎯 Learning Goals

By the end of this week, students will:

Understand Semantic Tags

Understand what semantic tags are and why they're used.

Structure Web Pages

Be able to structure a web page using <header>, <nav>, <main>, <section>, <article>, and <footer>.

Create Contact Forms

Learn how to create a simple contact form with input fields and a button.

Build Personal Website

Build a "Personal Website" with a structured layout and contact section.

🧠 Key Vocabulary

Term Meaning
Semantic HTML Tags that describe the meaning or purpose of content
<header> The top part of a page (logo, title, menu)
<nav> Section that contains navigation links
<main> The main content of the page
<section> A separate part or topic of a page
<footer> Bottom part of a page (copyright, contact info)
<form> Used to collect user input
<input> A field where users can type something
<label> Describes an input field
<button> Creates a clickable button

💬 Warm-up (10 minutes)

Ask:

  • "Who remembers what <a> does?"
  • "What's inside your profile page from last week?"
  • "Have you ever filled a form on a website (like a sign-up or feedback form)?"

Show an example of a website layout and point out:

"This top part is called a header. The links are in the nav. The main part is the content, and the bottom is the footer."

🧩 Step-by-Step Lesson

🏗️ Part 1: What is Semantic HTML? (15 minutes)

Explain:

"Semantic means meaningful. These tags help browsers and developers understand what each part of the page is for."

Show this non-semantic vs semantic comparison:

❌ Old way:

<div>
  <div>My Website</div>
  <div>Menu</div>
  <div>Content</div>
  <div>Footer</div>
</div>

✅ Semantic way:

<header>My Website</header>
<nav>Menu</nav>
<main>Content</main>
<footer>Footer</footer>

💬 Explain Simply:

"Using <header> instead of <div> tells the browser: 'This is the top of the page.' It's like labeling drawers — easier to find things later!"

🧱 Part 2: Create a Web Page Layout (20 minutes)

Step by step, create the following structure:

<!DOCTYPE html>
<html>
  <head>
    <title>My Personal Website</title>
  </head>
  <body>
    <header>
      <h1>Welcome to My Website</h1>
      <p>Created by Kareem</p>
    </header>

    <nav>
      <a href="index.html">Home</a> |
      <a href="about.html">About</a> |
      <a href="contact.html">Contact</a>
    </nav>

    <main>
      <section>
        <h2>About Me</h2>
        <p>Hi! I'm Kareem. I love coding, drawing, and video games.</p>
      </section>

      <section>
        <h2>My Projects</h2>
        <article>
          <h3>My Scratch Game</h3>
          <p>I made a game about jumping cats!</p>
        </article>
      </section>
    </main>

    <footer>
      <p>© 2025 Kareem's Website</p>
    </footer>
  </body>
</html>

Welcome to My Website

Created by Kareem

Home | About | Contact

About Me

Hi! I'm Kareem. I love coding, drawing, and video games.

My Projects

My Scratch Game

I made a game about jumping cats!

© 2025 Kareem's Website

💡 Explain:

  • <section> groups similar topics.
  • <article> is used for one piece of content like a story, post, or project.
  • <footer> always stays at the bottom.

Show how this structure looks visually:

  • Header at the top
  • Nav below it
  • Main content in the middle
  • Footer at the bottom

Encourage students to personalize their text and headings.

💌 Part 3: Creating a Simple Contact Form (20 minutes)

Explain:

"Forms let people send information to a website — like signing up, giving feedback, or saying hello."

Basic structure:

<form>
  <label>Your Name:</label><br>
  <input type="text"><br><br>

  <label>Your Email:</label><br>
  <input type="email"><br><br>

  <label>Your Message:</label><br>
  <textarea rows="4" cols="30"></textarea><br><br>

  <button>Send</button>
</form>









🧠 Explain Each Tag:

  • <label> — tells what the input is for.
  • <input> — where users type.
  • <textarea> — a bigger box for long text.
  • <button> — the clickable part!

Show how <br> adds line breaks for spacing.

💡 Tip for Kids:

"Right now, our form doesn't really send data anywhere — it just looks like a real contact form. Later we'll learn how to make it work!"

🎨 Combine Form with the Layout (10 minutes)

Add the contact form inside the <main> or a new <section>:

<section>
  <h2>Contact Me</h2>
  <form>
    <label>Name:</label><br>
    <input type="text"><br><br>

    <label>Email:</label><br>
    <input type="email"><br><br>

    <label>Message:</label><br>
    <textarea rows="4" cols="30"></textarea><br><br>

    <button>Send Message</button>
  </form>
</section>

Now their page feels real and interactive 🎉

🧩 Mini Project (10–15 minutes): "My Personal Website"

Goal: Build a simple website with:

  • A header and navigation
  • A main section about them
  • A project section
  • A contact form
  • A footer with copyright

Encourage creativity:

  • Add their profile image from Week 3
  • Use different headings and sections
  • Add real or pretend links to their "Projects" or "About Me" page

🎉 Show & Tell (5 minutes)

Ask:

  • "Which section did you enjoy creating most?"
  • "Why is using <header> and <footer> better than <div>?"
  • "Can you find your contact form in your code?"

Encourage others to say one nice thing about their classmates' pages 👍

🏡 Homework

  • Add one more <section> to their site (e.g., "My Hobbies" or "My Dreams").
  • Add one new <article> under "Projects" describing another thing they like.
  • Make sure every page includes <header> and <footer>.

🎯 Bonus Challenge (for fast learners)

Add a mailto link in the footer:

<a href="mailto:myemail@example.com">Email Me</a>

Add a "Back to Top" link:

<a href="#top">Back to Top</a>

Add a form placeholder:

<input type="text" placeholder="Enter your name">

⚠️ Common Mistakes

Mistake What Happens Fix
Forgot closing tags Layout breaks Always close </header>, </footer>, etc.
Missing <form> around inputs Form doesn't group Wrap all inputs inside <form>
Wrong nesting Elements misplaced Keep proper tag order (header → main → footer)
Too many <br> tags Messy layout Use sections instead

🧰 Teacher Tips

  • Draw a simple diagram on the board showing:
    [Header]
    [Nav]
    [Main Section]
    [Contact Form]
    [Footer]
  • Let students copy your base HTML, then personalize it.
  • If time allows, show a quick preview of how CSS will make it colorful next week!
  • Keep using simple, encouraging language — celebrate small wins like "You just built your first website layout!"
